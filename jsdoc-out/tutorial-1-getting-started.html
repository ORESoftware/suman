Tutorial: Getting Started with Suman

Getting Started with Suman

  1. $ npm install -g suman

  2. cd into the project where you want to use Suman to power your tests

  3. Run $ suman --init

Now you will see that you have some new files in your project. You have a suman.conf.js file at the root of your project. You also have a directory called suman at the root of your project which contains 3 files, suman.once.js, suman.order.js, suman.ioc.js. If you want to find out what these files are for, go to the Advanced Usage section from the home page, but for now, if you are brand new to suman you can ignore those.

Alright, let's run a test

Individual test suites can be run with either "$ node path/to/your-test.js" or "$ suman path/to/your-test.js", the result is the same.

To use the Suman runner, you would use "$ suman --rnr path/to/your-test.js" or "$ suman path/to/tests/folder"

If you point Suman at a folder, Suman must use the Suman runner to run the tests. If you point Suman at an individual test file, you the developer have the choice about whether to use the runner or not, using the --rnr flag.

The biggest advantage of using the runner with a single test file is that the runner can suppress your console.log/debugging output.
